# 对象存储安全规范

| 属性 | 值 |
| --- | --- |
| 状态 | active |
| 适用范围 | COS 或兼容对象存储的上传、下载、预览和 CDN 访问 |
| 来源 | COS 使用规范，经工作流消除实现矛盾并补充验证边界 |

## 强制规则

| ID | 规则 | 验证方式 |
| --- | --- | --- |
| `OBJ-001` | 用户上传前，服务端必须根据可信身份校验业务场景、对象归属、文件类型和大小 | 未授权和越权上传测试 |
| `OBJ-002` | 对象 Key 由服务端生成；客户端不得自行决定最终授权路径 | 修改 Key 后上传和提交测试 |
| `OBJ-003` | 临时凭据遵循最小权限、精确资源范围、最少 action 和短有效期；不得返回长期密钥 | STS 策略和过期测试 |
| `OBJ-004` | 每次授权记录主体、场景、对象 Key、约束和过期时间；业务提交时服务端再次核对 | 跨用户、跨场景和过期提交测试 |
| `OBJ-005` | 仅信任扩展名或 Content-Type 不足以确认文件安全；高风险文件按场景检查内容和大小 | 伪造类型、超限和恶意文件测试 |
| `OBJ-006` | 用户上传桶默认私有读写；公有资源与用户上传必须隔离，公有桶不得允许终端用户写入 | 桶策略和匿名访问测试 |
| `OBJ-007` | 私有对象通过服务端生成的短期签名 URL 或受控 CDN 访问，前端不拼接固定域名 | 过期、篡改和权限测试 |
| `OBJ-008` | 未完成提交的临时对象必须有可验证的清理方案；清理不得误删已生效对象 | 清理演练和引用一致性检查 |
| `OBJ-009` | 上传接口必须按用户和场景限流，并监控容量、失败率和异常类型 | 限流及告警测试 |

## 推荐流程

1. 客户端提交上传意图，包括场景、文件名、声明类型和大小。
2. 服务端使用登录态执行权限与配额校验，生成唯一对象 Key。
3. 服务端签发只允许该 Key 和必要上传动作的短期凭据，并保存待提交授权记录。
4. 客户端使用返回的 Key、区域、桶和类型直接上传对象存储。
5. 客户端回传对象 Key 和校验信息；服务端核对授权记录及对象元数据后落库。
6. 服务端使待提交记录失效；未提交对象由独立清理任务处理。

## 路径与访问

- Key 使用业务域、场景、可信主体和随机唯一标识组成，不含凭据和不必要的个人信息。
- 数据库存稳定对象 Key，不存短期签名 URL；访问域名和 CDN 策略由环境配置决定。
- 富文本引用需要解析 DOM 或结构化内容，不使用无法正确处理 HTML 的临时正则替换方案。
- 下载文件名使用标准 `Content-Disposition` 的 `filename*` 参数；Go 实现使用适合路径片段的百分号编码，不使用会把空格转换为 `+` 的 `url.QueryEscape`。

## 代码审查与验证

- 检查临时策略的 resource、action、condition、有效期和主体绑定。
- 检查服务端是否把客户端回传 Key、ETag 或 Content-Type 当作完整可信证明。
- 检查私有桶匿名访问、CDN 鉴权、签名续期和缓存泄露风险。
- 验证分片上传取消、重复提交、凭据过期、对象不存在和清理任务并发。
- verification 记录策略摘要、越权测试、访问过期测试和清理演练，不记录真实密钥。
