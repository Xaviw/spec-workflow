# 日志与可观测性规范

| 属性 | 值 |
| --- | --- |
| 状态 | active |
| 适用范围 | 服务端、任务消费者及调用外部服务的组件 |
| 来源 | 公司级日志规范 v1.1，经工作流精简 |

## 强制规则

| ID | 规则 | 验证方式 |
| --- | --- | --- |
| `LOG-001` | 请求入口、消息消费和定时任务必须建立可检索的关联 ID，并贯穿同步与异步调用链 | 端到端触发一次业务流程，使用同一 ID 检索入口、业务、数据访问和异步日志 |
| `LOG-002` | 跨 HTTP、RPC、消息和延时任务边界时必须传递关联 ID；已有父链路时不得无故重新生成 | 检查出站元数据和消费端上下文初始化 |
| `LOG-003` | 每条运行日志至少包含时间、级别、服务或模块、关联 ID、代码位置和可定位的业务事件 | 检查结构化字段或统一文本格式 |
| `LOG-004` | ERROR 必须包含原始错误和必要业务上下文，但不得输出凭据或完整敏感数据 | 制造可控错误并检查日志 |
| `LOG-005` | 外部调用必须记录目标标识、方法、结果、耗时和错误摘要；请求体、响应体和 Header 默认不完整记录 | 审查 HTTP/RPC 包装层及敏感字段过滤 |
| `LOG-006` | 高频循环、健康检查和批量数据不得逐条输出大量 INFO 日志 | 检查日志量和聚合策略 |
| `LOG-007` | 审计日志与运行日志职责分离；审计事件至少包含操作人、动作、对象、结果和时间 | 检查审计存储与运行日志边界 |
| `LOG-008` | 日志保留、切割和采集策略必须由部署环境明确配置，不得依赖无限增长的本地文件 | 检查运行配置和容量限制 |

## 日志级别

| 级别 | 使用场景 |
| --- | --- |
| DEBUG | 仅用于开发或临时诊断的详细信息，生产默认关闭 |
| INFO | 启动完成、关键业务状态、任务完成和外部调用摘要 |
| WARN | 重试、降级、接近阈值、慢调用等仍可继续的异常 |
| ERROR | 请求或任务失败、数据访问异常、外部依赖失败等需要处理的问题 |

不得用 INFO 隐藏真实错误，也不得用 ERROR 记录正常业务拒绝。SQL 日志是否启用、采样和脱敏由项目配置决定，不要求生产环境无条件输出完整 SQL 与参数。

## 敏感信息

- 禁止输出口令、访问令牌、刷新令牌、Cookie、私钥、SecretKey 和完整 Authorization Header。
- 手机号、证件号、银行卡、邮箱、地址及业务敏感金额按 `security.md` 处理。
- 日志完整性与安全规则冲突时，以 `POL-LOG-001` 和 `SEC-*` 规则为准。
- 日志脱敏优先使用字段级处理；框架级正则替换只能作为补充，不能代替明确选择日志字段。

## 设计检查

- 关联 ID 在哪里产生，采用什么字段名，跨边界如何传递？
- 异步任务没有父请求时如何生成稳定的任务关联 ID？
- 哪些事件属于运行日志，哪些属于审计记录？
- 外部调用、SQL 和大对象日志如何脱敏、采样和限制体积？
- 日志采集失败、磁盘写满或下游不可用时如何降级？

## 代码审查检查

- 业务代码是否创建了脱离父链路的空上下文。
- 错误是否只记录“失败”而缺少动作、对象和原始错误。
- 是否直接序列化请求、响应、环境变量或 Header。
- 循环、批处理、健康检查是否造成日志洪峰。
- 日志记录失败是否反过来破坏主业务路径。

## 验证证据

- 一次端到端请求或任务的关联 ID 检索结果摘要。
- 一次可控失败的 ERROR 字段和脱敏结果。
- 高频或批量场景的日志量检查。
- 日志采集、切割和容量配置的位置。

## 技术栈说明

Go、Java、Node.js、Python 应优先复用目标仓库已有日志与上下文设施。只有目标仓库缺少共享入口且任务明确要求改造时，才增加统一封装；不得从旧规范复制整套项目私有源码。
