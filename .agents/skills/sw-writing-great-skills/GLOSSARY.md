# 高质量技能术语表

这些术语用于编写和诊断 Skill。根目标是**可预测性**：让模型每次采用相同过程，而不是生成相同答案。

## 调用

### 可预测性

Skill 稳定约束执行过程的程度。成本和可维护性都应服从这一目标。

### 模型调用

模型能从常驻描述中发现并主动使用 Skill；用户仍可显式调用。它以持续的上下文占用换取自动发现能力。

### 用户调用

只有用户明确点名时才使用 Skill。在本工作流中，通过 `agents/openai.yaml` 的 `policy.allow_implicit_invocation: false` 实现。

### 描述

Skill 的机器可读入口。模型调用型描述同时说明能力和触发分支；用户调用型描述只需简短说明用途并限制为显式调用。

### 上下文指针

当前上下文中指向未加载材料的说明。指针的文字决定何时读取目标，链接地址本身不能代替触发条件。

### 上下文负载

模型调用型 Skill 的描述长期占用的令牌与注意力。只有自动发现确有价值时才承担它。

### 认知负载

用户记住并主动调用 Skill 的成本。用户调用型 Skill 越多，这项成本越高。

### 路由 Skill

帮助用户查找其他用户调用型 Skill 的入口。它列出选择条件，但不代替用户调用目标 Skill。

### 粒度

Skill 的拆分程度。按调用拆分会增加上下文负载；按用户调用拆分会增加认知负载；按顺序拆分则用于隔离会引发抢跑的后续步骤。

## 信息层级

### 步骤

模型按顺序执行的动作。每一步都应有完成条件；没有天然顺序的规则不必伪装成步骤。

### 参考

执行时按需查询的定义、事实、参数、示例或条件性说明。分支专用参考适合放到外部文件。

### 信息层级

内容按即时需要程度排列：正文步骤、正文参考、通过上下文指针读取的外部参考。最常用、最关键的内容靠前。

### 渐进披露

把只在特定分支需要的参考移到外部文件，使正文保持清晰。所有路径都需要的材料应留在正文。

### 共置

把同一概念的定义、规则和例外放在一起。共置解决信息分散，单一事实来源解决同义重复。

### 蔓延

Skill 即使内容都有效，也因过长而难以阅读和维护。通过渐进披露或按真实分支拆分解决。

## 行为引导

### 分支

Skill 的一种独立使用方式。不同分支会走不同路径，并需要不同上下文。

### 引导词

模型预训练中已有、能用少量令牌唤起整套行为的概念。优先使用成熟术语；自造词需要额外定义，通常得不偿失。

### 完成条件

判断一项工作是否结束的标准。清晰度防止过早完成，覆盖要求决定模型需要做多少查证和执行。

### 实际工作

模型在一个步骤内部完成的阅读、搜索、修改和核对。严格、可检查的完成条件会自然带来足够的实际工作。

### 后续步骤

当前步骤之后仍可见的动作。它们可能把注意力拉向“尽快结束当前步骤”。

### 过早完成

模型在当前步骤真正完成前转向后续步骤。先把完成条件写清楚；只有条件无法进一步明确且抢跑真实发生时，才拆分上下文。

### 否定引导

用禁令反复唤起不希望出现的行为。优先直接描述目标行为；无法正向表达的硬边界才保留禁令，并同时说明替代动作。

## 精简

### 单一事实来源

每个意思只有一个权威位置，行为变化时只需修改一处。

### 重复

同一意思在多处出现。它增加维护和令牌成本，还会让该意思获得不应有的注意力权重。

### 相关性

一行内容是否仍服务于 Skill 的目标。已经失效或只服务于未触发分支的内容不应留在正文。

### 沉积

旧内容因只增不减而逐层堆积。定期按相关性清理，避免失效规则继续影响模型。

### 无效指令

不改变模型默认行为的句子。判断标准是“删除它后行为是否会变化”；不会变化就删除。
