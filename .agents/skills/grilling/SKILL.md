# 结构化澄清决策

## 上下文契约

必读：当前请求中的访谈主题、已知事实、待决范围和完成标准。

按需读取：仅限查明当前事实所需的代码、配置、文档或工具输出。

初始禁止：无关任务、无关仓库、与当前决定无关的材料和大范围探索。

输出：已确认结论、阻塞项、显式未决项，以及用户对共同理解的确认。

## 流程

1. 将主题整理为决策树。能从环境查明的事实自行调查；只把判断、偏好和取舍交给用户。完成：事实已有证据，待决节点都需要用户选择。
2. 找出前置条件已确定且互不依赖的当前前沿。每轮编号询问全部前沿问题；每题给出推荐答案、主要影响和必要选项。问题高风险、回答冲突或仍有依赖时，每轮只问一题。完成：本轮问题都可以独立回答。
3. 等待用户回答，再更新决策树并重新计算前沿。未回答事项保持未决，不代答、不转成默认假设。完成：每个回答都已解决节点、产生后续节点或形成明确阻塞。
4. 重复步骤 2-3，直到前沿为空。总结已确认结论、阻塞项和显式未决项，请用户确认是否已达成共同理解。完成：所有分支均已解决或明确阻塞，且用户已经确认。

确认前不实施结论或改写下游产物；确认后将结果交还当前请求继续处理。
