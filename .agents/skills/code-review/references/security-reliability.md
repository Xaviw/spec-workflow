# 安全与可靠性审查

只检查审查范围实际触及的信任边界和运行路径。每项发现同时证明可达条件与影响，不把缺少某种通用防护本身视为漏洞。

## 输入与访问控制

- 跟踪外部输入进入 HTML、SQL/NoSQL、命令、GraphQL、文件路径、URL、对象合并和反序列化的位置，检查编码、参数化、规范化、允许列表和类型约束。
- 检查 XSS、注入、SSRF、路径穿越、原型污染和不安全反序列化是否具有真实输入路径。
- 检查新入口是否执行认证、授权、资源归属和租户隔离；身份、角色、租户和数据范围必须来自可信服务端状态。
- 检查跨域策略、安全响应头、错误响应和重定向是否暴露内部信息或放宽既有边界。

## 身份、令牌与敏感信息

- 检查会话固定、令牌过期，以及 JWT 的算法、签发方和受众验证；不要信任客户端声明的角色或归属。
- 检查凭据、令牌、个人信息和敏感载荷是否进入源码、客户端包、日志、错误、缓存或持久化数据。
- 检查脱敏是否发生在可信边界内，且不会因异常路径绕过。

## 数据与并发

- 检查多步写入的事务边界、失败回滚和部分成功状态；可重试操作应具有正确幂等语义。
- 检查读改写、先检查后执行和文件操作是否需要原子操作、唯一约束、乐观锁、悲观锁或条件更新。
- 检查计数、余额、库存、配额和状态转换在并发请求下是否丢失更新或重复执行。
- 检查缓存失效、读写顺序、事件排序、重复投递、分布式锁和领导者状态是否允许陈旧或冲突结果。

## 运行时与外部系统

- 检查外部调用的超时、取消、重试、退避和限流；重试不得放大非幂等副作用。
- 检查循环、递归、正则、缓冲区、集合、连接和文件句柄是否具有输入相关的资源上限。
- 检查异步或并发任务是否共享可变状态、泄漏后台任务或在进程退出后丢失关键工作。
- 检查同步 I/O、阻塞计算和逐项远程调用是否进入请求或其他高频路径。

## 加密与依赖

- 检查安全用途中的算法、密钥长度、随机数、IV、盐和认证加密；不要把编码或哈希误当作加密。
- 仅在依赖发生变化时检查来源、锁定、完整性、依赖混淆和已知漏洞；没有可验证风险时不因版本新旧报告问题。
